# TrajectAI — AI-Powered Career Assistant

A full-stack career platform that uses AI to help developers and professionals build resumes, generate cover letters, and practice interviews — all tailored to their industry, skills, and experience level.

---

## Motivation

Job searching is repetitive and time-consuming — tailoring a resume for each role, writing a fresh cover letter, practicing interview questions. I wanted to build a single tool that removes that friction by combining structured data (a user's skills, experience, industry) with generative AI to produce output that is actually specific to the person, not generic boilerplate.

The secondary goal was to build something production-grade: real auth, a real database, deployment, type-safe contracts end to end — not just a weekend demo.

---

## What I Built

### Resume Builder
- Rich markdown editor for every description field (bullets, numbered lists, bold/italic via `@uiw/react-md-editor`)
- Snapshot-based **undo/redo** with 700 ms debounce and a 50-snapshot ring buffer, implemented as a custom `useUndoable` hook
- 5 preset templates (Classic, Modern, Minimal, Executive, Fresh) that apply coordinated font/color/spacing styles
- Dynamic contact links — add or remove any link with a label+URL pair
- **PDF export** via `window.open` + `print()` — bypasses `html2canvas` which cannot parse Tailwind v4's `oklch()` CSS color functions
- ATS score and improvement suggestions generated by Gemini on demand

### Cover Letter Generator
- Select an existing resume + paste a job description → Gemini produces a tailored letter
- View, copy, and manage all generated letters

### Interview Prep
- Adaptive mock quiz engine: choose skill focus, category (technical / behavioral), difficulty (junior → staff), and question count
- **Skill tracks** — per-skill mastery system that auto-advances difficulty based on historical scores
- Performance chart showing score trend over time (Recharts)
- AI-generated improvement tips per quiz attempt

### Career Dashboard
- On login, Gemini analyses the user's industry + skills + experience and returns structured market data: salary ranges, demand level, job growth %, key trends, recommended skills
- Cached per user, refreshed weekly

## Architecture

```
personal-assistent/
├── backend/          # Express 5 + TypeScript API
│   ├── src/
│   │   ├── db/       # Drizzle ORM schema + Neon PostgreSQL client
│   │   ├── lib/      # Better-Auth setup
│   │   ├── modules/  # Feature modules (resume, cover-letter, interview, insight, user)
│   │   └── prompts/  # All Gemini prompt functions
│   └── index.ts      # Entry point (dotenv + listen)
├── frontend/         # React 19 + Vite SPA
│   └── src/
│       ├── routes/   # TanStack Router file-based routes
│       ├── components/features/  # Feature UI components
│       ├── lib/      # API client, auth client
│       └── context/  # Auth context
└── shared/
    └── types/api.ts  # Shared TypeScript interfaces used by both sides
```

**Key architectural decisions:**

| Decision | Why |
|----------|-----|
| Shared `types/api.ts` | Single source of truth for request/response shapes — breaking changes caught at compile time on both client and server |
| TanStack Router file-based routing | Type-safe `Link` and `navigate` — compiler rejects invalid routes or missing search params |
| Drizzle ORM | Thinner runtime, SQL-first, precise migrations — no magic |
| Better-Auth | Framework-agnostic, works with any Node server, session cookies with no JWT overhead |
| Gemini `gemini-2.5-flash-lite` | Fast enough for interactive features, cheap, reliable structured JSON output |
| `window.open` + `print()` for PDF | `html2canvas` cannot parse `oklch()` colors from Tailwind v4 — native browser print is more reliable |

## Key Technical Challenges

### 1. Undo/Redo Without a Library
The resume form has 15+ fields across 5 collapsible sections. I needed undo/redo that didn't snapshot on every keystroke and worked across section collapses, template switches, and AI-filled fields.

Solution: a `useUndoable` hook using two `useRef` values (`snaps[]` and `idx`) so snapshots don't cause re-renders, with a 700 ms debounce before committing to the history stack. A `skipRef` flag prevents the watcher from pushing a new snapshot when the change came from an undo/redo operation itself.

### 2. PDF Generation with Tailwind v4
Tailwind v4 uses CSS custom properties with `oklch()` colors. `html2canvas` resolves CSS variables at paint time then tries to parse the color value — it fails silently and outputs a broken canvas. Fix: open a `window.open` popup, write the resume element's `innerHTML` into it with a `<style>` tag containing `@page { size: A4 portrait; margin: 0 }`, then call `popup.print()`. Delegates rendering entirely to the browser's print engine which handles all CSS correctly.

### 3. Auth Transition Flicker with TanStack Router
After login, calling `router.invalidate()` re-ran `beforeLoad` guards using the *stale* React context (`isAuthenticated: false`) before the new session was reflected — causing a flash back to the login page. Fix: replace `router.navigate()` + `router.invalidate()` with `window.location.href = '/app'`. The hard reload starts a fresh React tree with the session cookie already set.

### 4. Nested `<button>` Hydration Error
The collapsible `Section` component used a `<button>` as its header toggle. When an AI action button was passed as the `action` prop, it rendered a `<button>` inside a `<button>` — invalid HTML. Fix: changed the outer element to `<div role="button" tabIndex={0}>` with a `onKeyDown` handler, with the action button as a sibling outside the clickable area.

### 5. Structured AI Output
All Gemini prompts instruct the model to return JSON-only with no markdown fences. The response is parsed with `JSON.parse()` and validated against expected shape. This is more reliable than asking for prose and extracting data from it — and gives consistent typed output for every AI feature.

## What I Achieved

- **End-to-end type safety** across the full stack via shared TypeScript interfaces — no runtime mismatches between client and server
- All AI features backed by a single Gemini client with **structured prompt engineering** that returns typed JSON, not freeform text
- **Production deployment** with CI/CD: GitHub push → Vercel auto-deploys frontend, Render auto-deploys backend
- Auth that works correctly across hard navigation boundaries without flickering — a subtle problem with SPA routers and session state
- A PDF export approach that is **actually reliable for modern CSS** — solving a real limitation of the most popular PDF libraries

## Running Locally

```bash
# Clone
git clone https://github.com/MohammadRaiyyan/personal-assistent
cd personal-assistent

# Backend
cd backend
cp .env.example .env        # fill in DATABASE_URL and GEMINI_API_KEY
npm install
npm run db:push             # create tables
npm run dev                 # :4000

# Frontend (new terminal)
cd frontend
cp .env.example .env        # set VITE_API_URL=http://localhost:4000
npm install
npm run dev                 # :3000
```
